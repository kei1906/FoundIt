FINAL PROJECT  
Software Design (CPE8)  
I. Title: Development of Web Application  
II. Objectives:  
1. Design and develop a functional web application tailored for university  
stakeholders.  
2. To create a concise SRS Document for a developed system.  
3. To identify the functional and non- functional part of a system.  
4. To forecast possible future enhancement of a developed system. 
III. Activity Task:  
Step 1: Choose a User Group  
• Select ONE of the following as your target users:  
o Faculty  
o Students  
o Student Organization  
• Discuss within your group what problems or needs your app will address.  
Step 2: Define the Project Scope  
• Brainstorm the main features of your web app.  
• Identify what your app can and cannot do.  
• Outline basic user roles (e.g., admin, regular user).  
Step 3: Draft the System Flowchart  
• Create a flowchart showing how users will interact with your system. • Use 
symbols to represent decisions, processes, inputs/outputs. • You may hand
draw the flowchart (engineering lettering) or use software.  
Step 4: Design the Interface  
• Sketch or wireframe your user interface.  
• Decide on page layouts, navigation, and user experience.  
• You may use Figma, Adobe XD, draw.io, or paper sketches.  
Step 5: Start Development  
• Set up your development environment.  
• Divide tasks among team members (frontend, backend, database). • Use 
HTML/CSS/JS or any framework/language of your choice (e.g., React, PHP,  
Django).  
Step 6: Build Core Features  
• Implement your app’s main functionalities.  
• Examples:  
o Login system  
o Dashboard 
o Forms for input  
o Data display (tables, charts, etc.)  
• Test each feature for bugs or issues.  
Step 7: Document Your Project  
Prepare your soft and hardcopy of written report (PDF & brochure) with the 
following: 1. Introduction  
2. Functionalities  
3. Flowchart  
4. Screenshots  
5. System Development (not included in the brochure)  
6. Limitations and Assumptions  
7. Future Enhancements  
Step 8: Prepare for Presentation  
• Summarize your app’s purpose and key features.  
• Demonstrate how it works (live demo).  
• Include your flowchart and system logic in your explanation.  
• Time limit: 5–10 minutes  
Step 9: Submit All Requirements  
by May 23, 2025, submit the following:  
• ✅ Web App (live link or offline demo)  
• ✅ Source Code (zip) & Soft copy of Report (PDF) in a flashdrive  
• ✅ Final Project Report (brochure)  
• ✅ Presentation slides (link in Google Sheets)
